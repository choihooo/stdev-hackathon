# Unicorn Slayer - Visual Design System

## Design Philosophy

"다크 보드룸에서 빛나는 스크린." 한밤중에 투자자 미팅 자료를 만드는 창업자의 그 느낌. 긴장감, 피곤함, 그래도 포기할 수 없는 집념. 모든 화면은 이 감정을 담아야 한다.

**키워드:** Dark Corporate Noir, Premium TCG, Pixel Art

**아트 방향:**
- 배경: 도시 야경 사진 (옅게 오버레이) → 스타트업 생태계의 밤
- 에셋: 16-bit 픽셀 아트 — 카드 아이콘, 적 캐릭터, UI 요소 모두 픽셀 통일
- 색감: 네온 대신 톤다운된 포인트 컬러. 차분하고 세련된 느낌
- 글로우/펄스 이펙트: 최소한. 은은한 테두리만 허용

## Typography

### 폰트 스택
```css

--font-mono:     'JetBrains Mono', 'SF Mono', monospace;

```

### 타입 스케일
```
Display (보스 이름, 타이틀):     28px / 700 / letter-spacing: -0.02em
Heading (섹션 제목):             20px / 700
Body (카드 설명, 로그):          14px / 400
Caption (상태 이상, 수치):       12px / 500
Micro (희귀도, 턴 카운트):       10px / 600 / letter-spacing: 0.05em / uppercase
Number (데미지, 수치):           --font-number, tabular-nums
```

### 적용 원칙
- 데미지 수치는 항상 `--font-number`로, 자릿수가 정렬되어야 함
- 카드 이름은 2줄 이하, 길면 줄임표
- 한글/영문 혼합 시 한글이 약간 더 크게 렌더링되므로 `font-size` 미세 조정 불필요 (Pretendard가 잘 처리함)

## Animation Principles

### 핵심 원칙
1. **모든 수치 변화는 눈에 보여야 한다.** 숫자만 바뀌는 게 아니라, 바가 줄어들고, 파티클이 튀고, 화면이 흔들려야 함.
2. **카드 액션은 물리적이어야 한다.** 드로우, 플레이, 버림 모두 물리적 움직임이 느껴져야 함.
3. **200ms 규칙.** 상태 변화 애니메이션은 150-250ms. 길면 답답함.
4. **연출은 건너뛸 수 있어야 한다.** 설정에서 애니메이션 속도 조절 가능.

### 타임라인 (전투 애니메이션)
```
카드 드로우:    180ms, ease-out, Y: +100 → 0
카드 사용:      250ms, spring, scale 1.0 → 1.2 → 0 (소멸)
카드 호버:      120ms, ease-out, translateY + scale
적 피격:        200ms, 화면 셰이크 X: ±4px, red flash
적 Phase 전환:  600ms, 배경 전환 + 오라 확산 + 사운드
힐 수치:        300ms, 초록 파티클 상승, 숫자 +N 팝업
데미지 수치:    250ms, 빨강 파티클 하강, 화면 미세 흔들림
방어 획득:      200ms, 시안 실드 이펙트, 바 즉시 증가
Burn Rate 차감: 400ms, HP바 천천히 감소, 텍스트 피드백
파산:           1.5s, 화면 암전 + 빨강 그래니언트 + 텍스트
승리:           2.0s, 골드 파티클 + 화면 밝아짐 + 결과
```

### Easing Functions
```css
--ease-card:     cubic-bezier(0.175, 0.885, 0.32, 1.275);  /* 바운스 */
--ease-damage:   cubic-bezier(0.6, -0.28, 0.735, 0.045);   /* 임팩트 */
--ease-smooth:   cubic-bezier(0.25, 0.46, 0.45, 0.94);     /* 부드러운 */
/* 스프링 이징은 Framer Motion의 spring 타입 사용 추천:
   <motion.div transition={{ type: "spring", stiffness: 300, damping: 20 }} /> */
```

### 파티클 이펙트
```
공격 적중:   딥 레드/앰버 스파크, 8-12개, 방사형, 300ms
회복:        포레스트 그린 방울, 6-8개, 상승, 500ms
방어 획득:   틸 헥사곤, 4-6개, 원형 확산, 400ms
버프/디버프: 타입 색상 원, 2-3개, 페이드인, 300ms
보스 Phase:  보스 색상 대형 파티클, 화면 전체, 1s
카드 드로우: 틸 꼬리, 카드 이동 경로, 180ms
```

## Component Specs


### 보스 Phase 전환 연출
```
1. 화면 암전 (300ms, opacity 0 → 0.6, #000)
2. 보스 이름 텍스트 크게 중앙 (400ms, fade-in)
   예: "PHASE 2 — 본격적 금리 인상"
3. 보스 오라 색상/크기 변경 (300ms)
4. 보스 보더 색상 변경 (Common → Intense)
5. 파티클 버스트 (타입 색상, 화면 중앙, 1s)
6. 화면 밝아짐 (300ms)
총 소요: ~1.5s
```

### 첫 5분 (First-Time Experience)
```
목표: 5분 안에 "이 게임, 계속 해보고 싶다" 느낌 주기

타임라인:
  0:00-0:05  로딩 화면 (로고 + 진행 바)
  0:05-0:15  메인 메뉴 등장 (페이드인 + 로고 애니메이션)
  0:15-0:30  "새 기업 만들기" 선택 → 창업자 선택 화면
  0:30-0:45  창업자 선택 → 짧은 인트로 서사 ("당신은 차고에서 시작합니다")
  0:45-1:00  첫 전투 진입 + 튜토리얼 시작
  1:00-3:00  첫 전투 (2-3턴, 쉬운 적, 튜토리얼 가이드)
  3:00-3:30  첫 승리 + 보상 선택 (카드 3장 중 1장)
  3:30-4:00  스테이지 맵 튜토리얼 ("다음 목적지를 선택하세요")
  4:00-4:30  상점/이벤트 (자연스러운 진행)
  4:30-5:00  두 번째 전투 (튜토리얼 없이, 독립 플레이)

튜토리얼 없이 게임을 시작한 경험자:
  - 메인 메뉴에 "튜토리얼 건너뛰기" 옵션
  - 건너뛰면 일반 난이도 첫 전투로 바로 진입
```

### 튜토리얼 시스템
```
원칙:
  1. 튜토리얼은 게임이다: 별도 모드가 아니라 쉬운 첫 전투
  2. 한 번에 하나만: 각 단계에서 하나의 개념만 가르침
  3. 하라면 하게 하라: 설명보다 행동. "드래그하세요"가 아니라
     카드에 가이드 핸들이 나타나고 드래그하면 자연스럽게 사용됨
  4. 스킵 가능: 언제든 "튜토리얼 끄기" 버튼 (상단 우측, 작게)

튜토리얼 스텝:
  Step 1: "이 카드를 적에게 드래그하세요"
    - 대상 카드에 시안 점선 테두리 + 떠오르는 애니메이션
    - 적 패널에 드롭 존 하이라이트 (시안 글로우)
    - 성공 시: "적에게 8 데미지!" + 파티클
  Step 2: "Focus가 부족하면 카드를 사용할 수 없습니다"
    - Focus 소모 시 Focus 바 차감 애니메이션
    - 비활성 카드 빨강 오버레이 + 잠금 아이콘
  Step 3: "턴을 종료하면 적이 공격합니다"
    - "턴 종료" 버튼 시안 펄스
    - 턴 종료 후 적 공격 + 화면 셰이크 + 데미지 팝업
  Step 4: "방어 카드로 피해를 줄이세요"
    - 방어 카드 하이라이트
    - 사용 후 실드 수치 증가 + 시안 이펙트
  Step 5: (자동) "이제 혼자서 진행하세요"
    - 튜토리얼 오버레이 페이드아웃
    - 이후 첫 전투까지 튜토리얼 UI 완전 제거

UI 요소:
  - 가이드 툴팁: 카드/버튼 근처에 말풍선 (반투명 var(--bg-tertiary) 배경)
  - 텍스트: 13px #e8e8f0, 최대 2줄
  - 화살표: 대상을 가리키는 시안 화살표
  - 비활성 영역: 가이드 외 영역 rgba(0,0,0,0.4) 오버레이
```

### 화면 전환 (Screen Transitions)
```
전환 유형별 타이밍 + 감정:

전환 원칙:
  - 페이드: var(--ease-smooth) 사용
  - 슬라이드: var(--ease-smooth) 사용
  - 등장 (scale): var(--ease-card) 사용 (바운스)
  - 퇴장: ease-in 사용 (가속하여 빠르게 사라짐)

메인 메뉴 → 전투:
  1. 화면 페이드아웃 (200ms, → #000, ease-in)
  2. 적 이름 + 아이콘 중앙 등장 (300ms, fade-in + scale 0.8→1.0, var(--ease-card))
  3. 전투 화면 페이드인 (300ms, var(--ease-smooth))
  총 소요: 800ms
  감정: "전투 시작, 긴장하라"

전투 → 스테이지 맵 (승리):
  1. 승리 연출 (골드 파티클 + 결과 요약, 2s)
  2. "계속" 버튼 또는 자동 (3초 후)
  3. 화면 밝아짐 (200ms) → 스테이지 맵 슬라이드인 (300ms, bottom→top)
  총 소요: ~2.5s (연출 포함)
  감정: "성공, 다음은 어디로?"

전투 → 결과 (패배):
  1. 패배 연출 (화면 암전 + 빨강 그라디언트, 1.5s)
  2. 결과 화면 페이드인 (500ms)
  감정: "좌절, 하지만 다시 하고 싶다"

스테이지 맵 → 전투:
  1. 선택한 노드 확대 (300ms, scale → 2.0)
  2. 화면 페이드아웃 (200ms)
  3. 전투 화면 페이드인 (300ms)
  총 소요: 800ms
  감정: "여기开始了, 집중"

스테이지 맵 → 상점:
  1. 노드 확대 (300ms)
  2. 상점 화면 슬라이드인 (300ms, right→left)
  감정: "휴식, 탐색"

스테이지 맵 → 이벤트:
  1. 노드 확대 (300ms)
  2. 이벤트 화면 페이드인 (300ms)
  감정: "무슨 일이 벌어질까?"

상점 → 스테이지 맵:
  1. "나가기" 선택
  2. 상점 슬라이드아웃 (300ms, left→right)
  3. 스테이지 맵 페이드인 (200ms)
  감정: "다시 출발"

이벤트 → 스테이지 맵:
  1. 결과 표시 (2s)
  2. "계속" 선택
  3. 화면 페이드 전환 (400ms total)
  감정: "결과 받아들이고 진행"

Act 전환:
  1. 현재 맵 페이드아웃 (400ms)
  2. "ACT 2 — 시리즈 B" 텍스트 중앙 등장 (500ms, 골드)
  3. 새 맵 슬라이드인 (400ms, bottom→top)
  총 소요: 1.3s
  감정: "새로운 챕터, 새로운 도전"
```

### 메타 진행 (Meta-Progression)
```
런 간 영구 요소:
  - 해금된 카드 풀: 다음 런에서 등장할 수 있는 카드 확장
  - 해금된 Relic 풀: 동일

  - 다음 런에서 새 콘텐츠 등장 가능

런 시작 시:
  - "새로운 카드가 카드풀에 추가되었습니다" 배너 (첫 드로우 시)
  - 이전 런에서 얻은 Relic이 상점에 등장 가능
```

### 로딩 상태 원칙
```
1. 스켈레톤 > 스피너: 카드/리소스 등 형태가 있는 요소는 스켈레톤 UI
2. 최대 로딩: 2초. 초과 시 경고 + 재시도 옵션
3. 게임 로딩 (초기 진입): 다크 배경 + 로고 골드 펄스 + 진행 바
4. 인게임 로딩: 스켈레톤 + 미세한 펄스. 화면 전환 없이 오버레이
5. 모바일: 로딩 시 스켈레톤 + "로딩 중..." 작은 텍스트 하단
```

### 오류 상태 원칙
```
1. 절대 빈 화면: 오류 시에도 배경+로고+상태바 유지
2. 오류 카드: 화면 중앙에 카드 프레임 형태의 오류 메시지
   - 배경: rgba(255,51,85,0.1), 보더 1px #ff3355
   - 아이콘: ⚠ 24px, 빨강
   - 텍스트: 14px #e8e8f0, 오류 내용
   - 버튼: 골드 "다시 시도" + 회색 "메인 메뉴"
3. 치명적 오류 (저장 불가): 전체 화면 오버레이 + 경고음
4. 복구 가능 오류: 토스트 + 자동 재시도 (3회, 1초 간격)
5. 네트워크 오프라인: Resource Bar 아래에 오프라인 배지 표시
   - 배경: rgba(255,51,85,0.15), 텍스트 "오프라인", 10px
```

### 빈 상태 원칙
```
1. 빈 상태는 기능이다: "아이템 없음"이 아니라 맥락+다음 행동 제공
2. 구조: 아이콘(32px) + 제목(14px bold) + 설명(12px #8888aa) + 행동 버튼
3. 애니메이션: 아이콘 미세한 떠오름 (float, 3s 무한)
4. 일러스트: 가능하면 커스텀 일러스트, 없으면 타입 컬러 아이콘
```

## Micro-interactions

### 카드 드래그 앤 드롭
```
1. 카드 롱프레스/클릭 → 카드 살짝 들림 (scale 1.05, shadow 깊어짐)
2. 드래그 → 카드가 커서를 따라감, 타겟 영역 하이라이트
3. 유효 타겟 위 → 타겟 녹색 글로우
4. 무효 타겟 위 → 카드 빨강 틴트
5. 드롭 → 카드 사용 애니메이션 또는 원위치 스프링백
```

### 적 공격 예고
```
의도 아이콘이 표시될 때:
  1. 기존 의도 페이드아웃 (150ms)
  2. 새 의도 슬라이드인 (200ms)
  3. 데미지 수치 카운트업 (100ms)
  4. 위험도 높으면 빨강 틴트 펄스
```

### 턴 전환
```
1. 패시브 정산 → 각 패시브 아이콘 순차 플래시 (100ms 간격)
2. Focus 충전 → Focus 수치 바 즉시 리필 + 시안 플래시
3. 드로우 → 카드가 덱 위치에서 손패로 하나씩 슬라이드인 (80ms 간격)
4. 턴 종료 시:
   a. 손패 카드 오른쪽으로 슬라이드 아웃 (100ms 간격)
   b. Burn Rate 차감 → HP바 천천히 감소 (300ms)
   c. 적 반격 → 화면 셰이크 + 데미지 팝업
```

## Accessibility

### 기본 원칙
- 색상 외 정보 보조: 버프/디버프를 색상만이 아니라 아이콘+텍스트로 구분
- 최소 대비: 텍스트 4.5:1 이상, 아이콘 3:1 이상 (위 컬러 시스템 참조)
- 폰트 스케일링: 최대 200%까지 레이아웃 유지
- 터치 타겟: 최소 44x44px (모바일)

### 키보드 네비게이션
```
포커스 표시기:
  - 기본: 2px solid var(--accent-cyan) 아웃라인 + 2px offset
  - 카드: 전체 카드 보더 시안 + 미세한 시안 글로우
  - 버튼: 인셋 시안 보더 + 텍스트 시안
  - 모바일: 포커스 표시기 숨김 (touch 기기는 시각 포커스 불필요)

탭 순서 (전투 화면):
  1. 손패 카드 (좌→우)
  2. End Turn 버튼
  3. 적 패널 (정보 확인용)
  4. Combat Log 토글
  5. 상태 이상 아이콘

키 바인딩:
  - Tab: 다음 포커스 가능 요소
  - Shift+Tab: 이전 요소
  - Enter/Space: 카드 사용, 버튼 클릭
  - Escape: 메뉴 열기, 오버레이 닫기
  - 1-5: 손패 카드 직접 선택 (숏컷)
  - E: End Turn (숏컷)
  - D: Combat Log 토글 (숏컷)

Skip Links:
  - 첫 번째 Tab: "주 콘텐츠로 건너뛰기" 링크 표시
  - 두 번째 Tab: "손패로 이동" 링크
```

### ARIA & Screen Reader
```
라이브 리전 (aria-live):
  - Combat Log: aria-live="polite", 새 엔트리 시 자동 읽음
  - Resource Bar 수치: aria-live="polite", 수치 변화 시 알림
    예: "Funds 75에서 60으로 감소"
  - HP 바: aria-live="polite", "적 HP 100에서 85로 감소"
  - Danger Gauge: aria-live="assertive", 위험도 75% 이상 시 경고

카드 역할 (role):
  - 손패 카드: role="button", aria-label="스케일업, 공격, 15 데미지, 비용 3"
  - 비활성 카드: aria-disabled="true"
  - 카드 호버 툴팁: aria-describedby로 상세 설명 연결

게임 상태 알림:
  - 턴 시작: "턴 3이 시작되었습니다. Focus 3/3"
  - 턴 종료: "턴을 종료합니다. 적이 공격 준비 중"
  - 승리: "전투 승리! 15 Funds 획득"
  - 패배: "전투 패배. 자금이 부족합니다"
  - 각 알림: aria-live="assertive", role="alert"

적 의도:
  - role="status", aria-label="적의 다음 행동: 15 데미지 공격"
```

### Reduced Motion (prefers-reduced-motion)
```
활성 상태 (기본):
  - 모든 애니메이션, 파티클, 셰이크, 트랜지션 활성

reduced-motion 감지 시:
  비활성화:
    - 화면 셰이크 (적 피격, 파산)
    - 파티클 이펙트 (공격, 힐, 방어, 보스 Phase)
    - 펄스 애니메이션 (위험 게이지, 보스 오라, 카드 글로우)
    - 배경 효과 (매트릭스 비, 노이즈)
    - 카드 드로우 슬라이드 애니메이션

  유지 (필수 정보 전달):
    - 수치 변화 (페이드만, 이동 없이)
    - HP/리소스 바 채움 변화 (즉시, 트랜지션 없이)
    - 카드 호버 translateY (기능적, 장식이 아님)
    - 토스트 표시/숨김 (페이드만)

  화면 전환:
    - 모든 전환 100ms로 단축
    - 슬라이드 → 페이드로 변경
    - 보스 Phase 전환: 텍스트만 표시 (오라/파티클 없이)

  사용자 설정:
    - 설정에서 "애니메이션 끄기" 수동 선택 가능
    - 시스템 설정 존중 + 사용자 오버라이드 가능
```

## Background & Asset System

### 게임 배경 (Background)
```
소스: 도시 야경 사진 (bg.jpeg)
적용: 전체 화면 배경으로 옅게 깔기

CSS 적용 방식:
  .battle-screen {
    background:
      linear-gradient(180deg, rgba(8,9,13,0.85), rgba(8,9,13,0.92)),
      url('/src/assets/bg.jpeg') center/cover no-repeat;
  }

오버레이: rgba(8,9,13,0.85~0.92) — 배경이 은은하게 보이되 UI 가독성 해치지 않는 수준
다른 화면도 동일 배경 + 다른 오버레이 톤 적용 가능:
  - 메인 메뉴: rgba(8,9,13,0.7) — 배경 조금 더 보이게
  - 전투: rgba(8,9,13,0.88) — 집중, UI 가독성 우선
  - 상점/이벤트: rgba(8,9,13,0.82) — 중간
```

### 픽셀 아트 에셋 가이드 (Pixel Art Asset Guide)

```
스타일: 16-bit 픽셀 아트 (SNES/Retro RPG 느낌)
해상도: 카드 아이콘 64x64px, 적 캐릭터 128x128px, UI 아이콘 32x32px
색상 팔레트: DESIGN.md 컬러 시스템 기반 (톤다운 악센트 컬러)
배경: 투명 (PNG)
아웃라인: 1px 짙은색 아웃라인 필수 (픽셀 아트 가독성)
앤티앨리어싱: 없음 (크리스피 픽셀)
```

### 에셋 목록
```
카드 아이콘 (64x64px):

  - 시리즈A 벤처캐피탈: 건물/은행 픽셀 아트
```

### 외부 의견에서 수용한 추가 설계 결정
```
1. 손패 제한: 최대 10장 (Slay the Spire 동일)
   - 10장 초과 시 초과 카드는 버림
2. 드로우 규칙 명확화:
   - drawPile 부족 시 discardPile 셔플 → drawPile
   - drawPile + discardPile 모두 비어있으면 드로우 불가
3. Burn Rate: MVP에서 유지. 수치 튜닝 필요 (시작값 1/턴)
4. 패시브 카드:
   - 턴 시작 시 자동 발동 (Focus 비용 없음)
   - hand에 있어야 발동 (다른 카드처럼 손에 들어옴)
   - 발동 후 버림 (지속 아님, 버프와 다름)
   - 중복 불가 (같은 패시브는 한 번만)
5. 리스크 카드:
   - 강력한 효과 + 부작용 (예: 30 데미지 + 자신 10 데미지)
   - Focus 비용 있음
```

## Game Design Decisions (UI 영향)

### 세이브/로드
```
저장소: LocalStorage + IndexedDB
  - LocalStorage: 설정 (사운드 on/off, 애니메이션 속도, 난이도)
  - IndexedDB: 게임 상태 (런 데이터, 덱, 인벤토리, 해금)
  - 자동 저장: 매 턴 종료 후, 전투 승리 후, 상점/이벤트 선택 후
  - 저장 표시: Resource Bar 좌측에 "저장됨" 시안 도트 (1초 후 fade-out)
  - 로드: 메인 메뉴 "이어하기" → 최근 세이브 선택
  - 세이브 슬롯: 3개 (해커톤 데모용), 향후 확장 가능
  - 오류 시: "저장에 실패했습니다" 토스트 + 재시도 버튼
```

### 사운드 기본값
```
첫 방문:
  - 사운드 ON 기본
  - 첫 화면 로드 시 짧은 시작 효과음 (200ms, 낮은 볼륨)
  - Resource Bar에 🔊 아이콘으로 음소거 토글 표시
  - 음소거 시: 🔇 아이콘, 모든 효과음 BGM 정지
  - 설정에서 BGM/효과음 개별 볼륨 조절 (0-100%)
  - 해커톤: 볼륨 낮춤 기본 (50%)으로 시작 권장
```

### 카드 드로우 규칙
```
셔플: 덱이 비어있으면 버림 패를 셔플하여 새 덱으로
드로우 순서: 완전 랜덤 (Fisher-Yates 셔플)
초기 손패: 5장 (첫 턴)
매 턴 드로우: 2장 (기본), 카드/Relic 효과로 증감 가능
드로우 애니메이션: 덱 위치(좌측)에서 손패로 슬라이드인 (80ms 간격)
덱 부족: 남은 카드만 드로우, "덱이 거의 비었습니다" 경고 (3장 이하)
```

### 게임 오버 조건
```
패배 조건 (OR):
  1. HP ≤ 0: Burn Rate에 의한 자연 감소 또는 적 공격으로 HP 고갈
     → "자금이 완전히 고갈되었습니다" 패배 메시지
  2. (선택) 턴 수 초과: 보스전 30턴 제한 초과 시
     → "시간이 너무 오래 걸렸습니다" 패배 메시지

HP 시스템:
  - HP = 기업의 자금 (시작: 100)
  - Burn Rate = 매 턴 차감되는 고정 비용
  - 적 공격 = 일회성 데미지
  - 회복 카드 = 자금 유입
  - HP 바가 빨강일 때 (25% 이하): 화면 가장자리 빨강 비네트
    + "자금 부족" 경고 토스트 매 턴
```

### 보스 Phase 전환 트리거
```
일반 원칙:
  - 모든 보스는 2 Phase (기본)
  - Phase 전환: HP 50% 이하 시 트리거
  - 전환 시 HP 회복 없음, 패턴만 변화

보스별 차이:
  - Act 1 보스: Phase 2에서 공격력 +30%, 새 패턴 1개 추가
  - Act 2 보스: Phase 2에서 방어력 획득, 매 턴 버프
  - Act 3 보스: Phase 2에서 HP 20% 회복 + 공격력 x2
  - (각 보스 상세 설계는 게임 디자인 문서로 분리)

UI 표현:
  - Phase 전환 연출: 기존 스펙 (디자인 시스템 > 보스 Phase 전환 연출)
  - Phase 2 돌입 시: "PHASE 2 — {보스별 텍스트}" 중앙 텍스트
  - 보스 HP 바 색상 변화: 빨강 → 보라 그라디언트
```

### 상점 리롤
```
리롤 규칙:
  - 기본 리롤: 1회 무료 (런당)
  - 추가 리롤: 없음 (해커톤 버전)
  - 향후: 20 Funds에 1회 추가 리롤 가능
  - 리롤 버튼: 상점 카드 섹션 우측 상단
    - 배경: var(--bg-elevated)
    - 아이콘: 🔄 + "리롤 (1/1)" 텍스트
    - 사용 후: 회색 + "리롤 (0/1)" + 비활성
    - 애니메이션: 기존 카드 슬라이드아웃 → 새 카드 슬라이드인 (400ms)
```
